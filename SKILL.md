# 机器人控制 Skill（OpenClaw）

本 Skill 面向已经在本地部署并运行 OpenClaw 的场景，用于把用户的自然语言机器人操作请求转换为对 `scripts/validate.py` 的实际调用。

当前 Skill 严格以现有脚本能力为准，不扩展脚本中尚未实现的接口，不假设额外环境变量，不引入代码中不存在的配置方式。

> ⚠️ 安全提醒：本 Skill 会触发现实世界中的机器人动作、夹爪动作、急停、下装移动工具运动与控制权释放。任何涉及运动、夹爪、急停、移动工具运动、关闭机器人的请求，都必须在现场完成安全确认后才执行。

## Goal（目标）

调用 `scripts/validate.py` 完成当前已封装好的机器人基础控制、下装移动工具控制与结束控制操作，并输出真实执行命令与真实脚本结果，确保操作可追溯、可复现、可审计。

## 当前能力范围

本 Skill 只覆盖以下命令：

### 机器人基础控制
- `init_all` - 完整初始化流程（获取 token、重置错误、初始化机器人）
- `grip_open` - 打开夹爪
- `grip_close` - 关闭夹爪
- `grip_position` - 夹爪移动到指定位置
- `perform` - 执行目标区域动作流程
- `safe` - 回到安全位
- `shutdown` - 急停机器人
- `camera` - 获取当前定位相机图片
- `status` - 查看本地状态

### 下装移动工具控制（新增）
- `agv_goto <location>` - 下装移动工具运动到指定站点（如 LM1, LM7）
- `vehicle_stop` - 停止下装移动工具运动
- `vehicle_home` - 下装移动工具回零
- `vehicle_location` - 获取当前下装移动工具位置

### 系统控制
- `close_robot` - 关闭机器人并重置本地状态（反初始化、销毁 token、清空状态）

除以上命令外，其他动作一律视为“当前 Skill 未覆盖”。

## When to use（何时触发）

当用户明确要求执行以下一种或多种机器人操作时触发：

### 基础控制
- 初始化机器人
- 测试初始化链路
- 打开夹爪
- 关闭夹爪
- 将夹爪移动到指定位置
- 执行某个区域的 Perform 动作
- 回到安全位
- 获取当前定位相机图片
- 急停机器人

### 下装移动工具控制（新增）
- 下装移动工具运动
- 移动工具运动到某个站点
- AGV 运动到 LM1/LM7 等站点
- 停止移动工具运动
- 移动工具回零
- 获取移动工具当前位置
- 查询移动工具位置

### 系统控制
- 关闭机器人
- 结束机器人控制
- 释放控制权并清空本地状态
- 查看本地状态

## Do not use（不要用）

以下情况不要触发本 Skill：

- 用户只是想了解“这段代码是什么意思”“原理是什么”
- 用户只是讨论机器人架构、实验方案、能力边界
- 用户要求执行当前脚本未实现的动作（如复杂的轨迹规划、多步实验流程）
- 用户没有给出明确动作目标，却要求直接操作机器人
- 用户要求绕过 token、控制权或安全确认
- 用户要求一次性生成长篇实现代码，而不是执行已有脚本
- 用户要求控制当前脚本未封装的设备

## Inputs to collect（需要收集的信息）

### 一、必须先识别动作类型

必须先把用户意图映射为以下某一个命令：

#### 基础控制
- `init_all`
- `grip_open`
- `grip_close`
- `grip_position`
- `perform`
- `safe`
- `shutdown`
- `camera`
- `status`

#### 下装移动工具控制（新增）
- `agv_goto`
- `vehicle_stop`
- `vehicle_home`
- `vehicle_location`

#### 系统控制
- `close_robot`

### 二、按动作收集参数

#### 1. `grip_position`
必须收集：

- `value`：夹爪目标位置值（整数）

#### 2. `perform`
必须收集：

- `target`：目标区域名称

可选参数：

- `vel`：速度，默认 `30`
- `acc`：加速度，默认 `30`
- `wait`：是否等待执行完成，默认 `1`

#### 3. `safe`
可选参数：

- `target`：安全位名称，默认 `Safe`

#### 4. `camera`
可选参数：

- `out`：图片保存路径；如果用户没要求保存，则不传

#### 5. `agv_goto`（新增）
必须收集：

- `location`：站点名称，如 `LM1`、`LM7`、`LM10` 等

#### 6. 其他命令
`vehicle_stop`、`vehicle_home`、`vehicle_location`、`init_all`、`grip_open`、`grip_close`、`shutdown`、`close_robot`、`status` 不需要额外参数

### 三、无需收集的内容

以下内容当前脚本中已固定，不需要向用户收集：

- `BASE_URL`
- `API_TIMEOUT`
- `STATE_FILE`
- 环境变量
- token 文件路径

## 安全门禁（执行前必须检查）

对以下命令，在执行前必须确认现场安全：

### 基础控制
- `grip_open`
- `grip_close`
- `grip_position`
- `perform`
- `safe`
- `shutdown`

### 下装移动工具控制（新增）
- `agv_goto` - 移动工具运动可能引起位置变化
- `vehicle_stop` - 停止运动
- `vehicle_home` - 回零动作可能产生运动

### 系统控制
- `close_robot`

安全确认至少包含：

1. 危险区域内无人
2. 机器人运动范围内无障碍物
3. 夹爪动作不会夹到人或设备
4. 移动工具运动路径上无障碍物
5. 急停可触达
6. 用户明确要执行真实动作，而不是仅测试解释逻辑

如果用户没有明确真实执行意图，或无法确认现场安全，则停止，不执行任何现实动作。

对于只读操作（`camera`、`status`、`vehicle_location`），可以适当放宽安全要求，但仍需提醒用户。

## Procedure（执行流程）

### 第一步：识别命令

根据用户自然语言，把请求映射到当前脚本支持的命令之一。

映射规则如下：

#### 基础控制
- “初始化机器人”“测试初始化链路” → `init_all`
- “打开夹爪” → `grip_open`
- “关闭夹爪” → `grip_close`
- “夹爪移动到 300”“夹爪开到 500” → `grip_position`
- “执行 Perform 到 A 区域”“在 Safe 区域执行动作” → `perform`
- “回安全位” → `safe`
- “获取相机图片”“拍一张当前图像” → `camera`
- “急停机器人” → `shutdown`

#### 下装移动工具控制（新增）
- “下装移动工具运动到 LM1”“AGV 运动到 LM7”“移动工具到站点” → `agv_goto`
- “停止运动”“移动工具停止” → `vehicle_stop`
- “移动工具回零”“下装移动工具回零” → `vehicle_home`
- “获取移动工具位置”“查看当前位置”“移动工具在哪” → `vehicle_location`

#### 系统控制
- “关闭机器人”“结束机器人控制”“释放控制权” → `close_robot`
- “查看当前状态”“机器人状态” → `status`

### 第二步：补齐参数

如果命令需要参数，则按以下规则处理：

#### `grip_position`
- 缺少 `value` 时，不执行，直接提示缺少参数

#### `perform`
- 缺少 `target` 时，不执行，直接提示缺少参数
- `vel` 缺省时使用 `30`
- `acc` 缺省时使用 `30`
- `wait` 缺省时使用 `1`

#### `safe`
- `target` 缺省时使用 `Safe`

#### `camera`
- 如果用户没有要求保存，不传 `--out`

#### `agv_goto`（新增）
- 缺少 `location` 时，不执行，直接提示缺少站点名称
- 常见站点格式：LM1、LM2、LM3、LM7、LM10 等
