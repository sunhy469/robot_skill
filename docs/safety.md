# 安全门禁与故障排查（合并版）

## A. 安全门禁（强制）

以下命令属于“真实运动/状态改变”，执行前必须先做门禁确认：

- 全部 `robot_*`（含运动、停机）
- 多数 `action_*`（夹爪、AGV、vehicle、外围设备）
- 多数 `command_*`
- 操作型 `script_api_*`

### 执行前四项确认

1. 危险区域无人；
2. 轨迹/路径无障碍、无碰撞风险；
3. 急停可达且可用；
4. 用户明确授权“允许真实执行”。

若任一项不满足：
- 禁止执行；
- 仅返回风险说明与补救建议。

---

## B. 故障排查

### 常见失败类型

1. 参数缺失
   - 处理：列出缺失字段并给出可执行模板。

2. JSON 解析失败（如 `steps`）
   - 处理：指出具体字段，给出合法 JSON 示例。

3. 数值转换失败（如 `velocity` / `acceleration`）
   - 处理：提示必须为可转 `float` 的数值。

4. 网络失败
   - 处理：返回网络错误，建议检查机器人服务连通性后重试。

5. 业务失败
   - 处理：原样回传业务错误；必要时建议先执行 `init_all` 或重新申请 token。

---

## C. 回退动作优先级

1. `robot_stop_motion`
2. 必要时 `robot_shutdown`
3. 结束后 `authority_consume`
